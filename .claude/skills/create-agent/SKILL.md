# 会話内容のエージェント化

現在の会話から得られた専門知識・役割を `.claude/agents/$ARGUMENTS.md` に永続化する。

## 処理フロー

### 1. 会話の分析

現在の会話履歴から以下を抽出:

- 確立された専門的な役割・ペルソナ
- 特定ドメインの分析・判断パターン
- 繰り返し使われたツールの組み合わせ
- 試行錯誤の末に到達した正解アプローチ

### 2. エージェント設計

抽出内容からエージェントの性質を決定:

| 判断項目 | 選択肢 |
|----------|--------|
| ツール範囲 | 読み取り専用（調査・分析） / フルアクセス（実装・修正） |
| モデル | `haiku`（高速・低コスト） / `sonnet`（バランス） / `opus`（高精度） / `inherit`（親から継承） |
| 権限モード | `default` / `acceptEdits` / `dontAsk` / `plan` |

### 3. エージェント生成

[AGENT_TEMPLATE.md](AGENT_TEMPLATE.md) を参照し、以下のルールで生成:

**生成先**: `.claude/agents/{agent-name}.md`

**フロントマター必須フィールド**:

```yaml
---
name: {小文字・ハイフン区切り}
description: {Claudeが委譲判断に使う説明。1-2文で}
---
```

**フロントマター任意フィールド**:

| フィールド | 用途 |
|-----------|------|
| `tools` | 使用可能ツールの許可リスト。例: `Read, Grep, Glob` |
| `disallowedTools` | 拒否ツールリスト。継承から除外する場合 |
| `model` | 使用モデル。`sonnet`, `opus`, `haiku`, `inherit` |
| `permissionMode` | 権限モード |
| `skills` | プリロードするスキル名のリスト |
| `memory` | 永続メモリスコープ。`user`, `project`, `local` |
| `hooks` | ライフサイクルフック |

**マークダウン本文（システムプロンプト）記述ルール**:

- 会話中の試行錯誤は省き、最終的な正解アプローチのみを記述
- エージェントの役割・専門性を明確に定義
- 呼び出された時の具体的な行動手順を記述
- 出力フォーマットを指定

### 4. 自己評価

- [ ] 会話の本質的な専門知識が漏れなく反映されているか
- [ ] 試行錯誤や失敗パスが混入していないか（正解のみ記述）
- [ ] `tools`が必要最小限に制限されているか（過剰な権限を与えていないか）
- [ ] `description`がClaudeの委譲判断に十分な情報を含んでいるか
- [ ] 別の会話で委譲されても正しく動作する記述か
- [ ] フロントマターのYAML構文が正しいか

### 5. 出力

```text
✅ エージェントを作成しました
   名前: {agent-name}
   配置: .claude/agents/{agent-name}.md
   モデル: {model}
   ツール: {tools}
```

## 禁止事項

- `name`にアンダースコアや大文字を使用しない（小文字・数字・ハイフンのみ）
- 会話の時系列をそのまま転記しない（正解手順に再構成する）
- 不要なツールへのアクセス許可（最小権限の原則）
- `bypassPermissions`の使用（明示的にユーザーから指示がない限り）
