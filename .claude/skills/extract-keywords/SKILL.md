# キーワード抽出スキル

`gen/data_source/` 配下の収集データを読み込み、アプリケーション設計に活かせるキーワードとトレンドを抽出して `keyword.json` を生成する。

## 対象ディレクトリの決定

- `$ARGUMENTS` が指定されている場合: `gen/data_source/$ARGUMENTS/` を対象とする
- 指定がない場合: `gen/data_source/` 配下で最新（名前の辞書順で最後）のサブディレクトリを対象とする

対象ディレクトリの確認:

```
!`ls -1 gen/data_source/ | tail -5`
```

## 手順

### 1. データ読み込み

対象ディレクトリ内の全データファイルを読み込む。以下の形式に対応:

- **NewsAPI** (`news.json`): `data.articles[].title` と `data.articles[].description`
- **YouTube** (`youtube.json`): `data.items[].snippet.title` と `data.items[].snippet.description`
- **ユーザー提案** (`user_proposal.md`): Markdownテキスト全体。`##` 見出し単位でセクション分割して読み込む
- **任意のテキストファイル** (`*.md`, `*.txt`): ディレクトリ内の上記以外の `.md` / `.txt` ファイルも自動的に読み込む。`.md` は `##` 見出し単位で分割、`.txt` はファイル全体を1エントリとして扱う

### 2. キーワード・トレンド抽出

読み込んだ記事群を分析し、キーワードとトレンドを抽出する。

抽出は**プロンプトで指定された連想モード設定**に従って行う:

#### 連想モード無効の場合

収集データに**直接記載されている**キーワードのみを抽出する:
- 記事タイトル・本文に明示的に登場する固有名詞・技術用語・トレンド名
- 複数記事で繰り返し言及されるテーマ
- 連想や推測による追加は行わない

#### 連想モード有効の場合

まず直接抽出を行い、さらに**連想ゲーム方式**で関連キーワードを追加する。

**重要: ニュースの要約ではなく「連想ゲーム」方式で抽出すること。**

- 記事の内容そのものではなく、そこから**派生する**ビジネス機会・ユーザーニーズ・技術トレンドに着目
- 「この出来事があるなら、こんなアプリが求められるのでは？」という思考
- 複数の記事を横断して見えてくる大きなトレンドやテーマを特定

連想の深さはプロンプトの `depth` 指定に従う:

| 深さ | 連想の範囲 | 例（元データ: 「AI画像生成の著作権問題」） |
|------|-----------|------------------------------------------|
| **shallow** | 1段階。直接的な関連語・類義語 | → 画像認識、コンテンツモデレーション、デジタル権利管理 |
| **moderate** | 2段階。関連分野・応用領域まで | → クリエイター支援ツール、AI倫理チェッカー、ライセンス自動管理 |
| **deep** | 3段階以上。異分野の組み合わせ・創造的飛躍 | → 「AIと人間の共作」認証プラットフォーム、創作プロセス可視化SNS |

連想で追加されたキーワードには `"source": "association"` を付与し、直接抽出されたものには `"source": "direct"` を付与する。

### 3. keyword.json の生成

対象ディレクトリ内に `keyword.json` を以下のフォーマットで書き出す:

```json
{
  "generated_at": "ISO 8601形式のタイムスタンプ",
  "source_dir": "対象ディレクトリ名",
  "association_mode": {
    "enabled": true,
    "depth": "moderate"
  },
  "keywords": [
    {
      "word": "キーワード",
      "category": "カテゴリ（ライフスタイル, ビジネス, テクノロジー, 教育, 健康 等）",
      "relevance_score": 0.0,
      "source": "direct または association",
      "source_articles": ["元になった記事タイトル"],
      "app_design_hints": ["このキーワードから連想されるアプリの方向性"]
    }
  ],
  "trends": [
    {
      "theme": "トレンドテーマ",
      "related_keywords": ["関連キーワード"],
      "potential_directions": ["このトレンドから考えられるアプリの方向性"]
    }
  ]
}
```

- `association_mode`: プロンプトで指定された連想モード設定を記録
- `source`: `"direct"`（データから直接抽出）または `"association"`（連想で追加）。連想モード無効時は全て `"direct"`

### 4. 結果報告

生成完了後、以下を報告:
- 対象ディレクトリ名
- 読み込んだ記事数
- 抽出したキーワード数とトレンド数
- 特に注目度の高いキーワード/トレンドのサマリー（上位3件程度）

## 注意事項

- `relevance_score` は 0.0〜1.0 の範囲で、アプリ設計への有用度を示す
- キーワードは最低5個、トレンドは最低2個を抽出する
- 日本語の記事・英語の記事が混在する場合があるが、出力は日本語で統一する
- `keyword.json` は既存ファイルがあれば上書きする
