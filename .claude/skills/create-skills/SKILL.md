# 会話内容のスキル化

現在の会話から得られた知識・手順を `.claude/skills/$ARGUMENTS/SKILL.md` に永続化する。

## 処理フロー

### 1. 会話の分析

現在の会話履歴から以下を抽出:

- 確立された手順・ワークフロー
- 発見したパターン・規約
- 試行錯誤の末に到達した正解アプローチ
- ドメイン固有の知識

### 2. スキルタイプの判定

抽出内容に応じて最適なタイプを選択:

| タイプ | 判定基準 |
|--------|----------|
| タスク型 | 手順・ワークフローが主体 |
| リファレンス型 | 規約・パターン・知識が主体 |

### 3. スキル生成

[SKILL_TEMPLATE.md](SKILL_TEMPLATE.md) を参照し、以下のルールで生成:

**生成先**: `.claude/skills/{skill-name}/SKILL.md`

**フロントマター必須フィールド**:

```yaml
---
name: {小文字・ハイフン区切り、最大64文字}
description: {1文でスキルの用途を説明}
disable-model-invocation: true
---
```

**フロントマター任意フィールド**:

| フィールド | 用途 |
|-----------|------|
| `argument-hint` | 引数ヒント表示。例: `"[target]"` |
| `allowed-tools` | 使用ツール制限。例: `Read, Grep, Glob` |
| `context: fork` | サブエージェントで分離実行 |
| `agent` | `context: fork`時のエージェントタイプ |

**コンテンツ記述ルール**:

- 会話中の試行錯誤は省き、最終的な正解手順のみを記述
- `$ARGUMENTS` で呼び出し時の引数を参照可能
- `` !`command` `` で実行前にシェルコマンド結果を埋め込み可能
- 500行以下に収める。超える場合はサポートファイルに分離し `SKILL.md` から参照

### 4. 自己評価

- [ ] 会話の本質的な知見が漏れなく反映されているか
- [ ] 試行錯誤や失敗パスが混入していないか（正解のみ記述）
- [ ] 別の会話・コンテキストで読んでも再現可能な記述か
- [ ] フロントマターのYAML構文が正しいか
- [ ] 500行以下か

### 5. 出力

```text
✅ スキルを作成しました
   名前: {skill-name}
   配置: .claude/skills/{skill-name}/SKILL.md
   呼び出し: /{skill-name}
```

## 禁止事項

- `name`にアンダースコアや大文字を使用しない
- 会話の時系列をそのまま転記しない（正解手順に再構成する）
- 500行超えの`SKILL.md`
