# 06: パイプライン統合・一括実行

## 概要

データ収集 → キーワード抽出 → 要件生成の3ステップを一括で実行するコマンドを提供する。個別コマンドの組み合わせを自動化し、1コマンドでデータ収集から要件生成までを完結させる。

## 前提

- `02_data-collection.md`、`03_keyword-extraction.md`、`04_requirements-generation.md` が完了していること

## 必要な作業

### 一括実行コマンド

- 実行コマンド: `pnpm run pipeline` または `tsx scripts/pipeline.ts`
- 処理フロー:
  1. `collect` - データ収集
  2. `extract` - キーワード抽出
  3. `generate` - 要件生成
- 各ステップの成功/失敗を追跡し、途中失敗時に適切にレポート
- オプション:
  - `--skip-collect`: 既存データを使ってキーワード抽出から開始
  - `--skip-extract`: 既存キーワードを使って要件生成のみ実行
  - `--source`: 使用する `data_source` ディレクトリの指定

### 実行ログ

- 各ステップの開始・終了時刻、処理件数をコンソールに出力
- 最終結果サマリー: 「{N}件の記事から{M}個のキーワードを抽出し、アプリ案「{app_name}」を生成しました → requirements/{app_name}/」

## 完了条件

- `pnpm pipeline` で収集→抽出→生成が順番に実行される
- 途中のステップで失敗した場合、適切なエラーメッセージが表示される
- `--skip-*` オプションで途中から実行できる
