# 04: 要件生成コマンド

## 概要

抽出したキーワード（`keyword.json`）を元に、LLM（Claude API）を使ってアプリケーションのアイデアを生成し、詳細な要件定義を `requirements/{app_name}/` に出力する。マネタイズ方法や運用方式まで含む包括的な要件を生成する。

## 前提

- `03_keyword-extraction.md` が完了し、`keyword.json` が存在すること
- Claude API（`ANTHROPIC_API_KEY`）が利用可能なこと

## 必要な作業

### 要件生成ロジック（LLM連携）

- `keyword.json` を入力として、以下をLLMに生成させる:
  1. アプリケーション案（名前・コンセプト）
  2. ターゲットユーザー
  3. 主要機能一覧
  4. 各機能の詳細仕様
  5. マネタイズ方式
  6. 運用・インフラ方針
  7. 技術スタック提案

### プロンプト設計

- 段階的な生成を行う（1回のプロンプトで全てを出すのではなく、段階的に深掘り）
  1. **アプリ案生成プロンプト**: キーワード群 → アプリコンセプト + 概要
  2. **機能分解プロンプト**: アプリ概要 → 機能一覧 + 各機能の概要
  3. **機能詳細プロンプト**: 各機能 → 詳細仕様（画面構成、データモデル、API設計等）

### 出力ファイル構造

README記載のディレクトリ構造に従い、以下を生成:

```
requirements/{app_name}/
├── _source_info.md          # 使用したdata_sourceやキーワードの整理
├── overview.md              # アプリ全体の概要
└── features/
    ├── 01_feature_name.md   # 機能1の詳細仕様
    ├── 02_feature_name.md   # 機能2の詳細仕様
    └── ...
```

### overview.md テンプレート

```markdown
# {アプリ名}

## コンセプト
{1-2文で簡潔に}

## ターゲットユーザー
{想定ユーザー層}

## 機能一覧
| ID | 機能名 | 概要 | 優先度 |
|----|--------|------|--------|
| 01 | ...    | ...  | 高/中/低 |

## マネタイズ
{収益化の方針}

## 技術スタック（提案）
{推奨する技術構成}

## 運用方針
{インフラ・デプロイ・監視等}
```

### features/{feature_id}_{feature_name}.md テンプレート

```markdown
# {機能名}

## 概要
{この機能が解決する課題と提供する価値}

## 画面構成
{画面の説明、ワイヤーフレーム的な記述}

## ユーザーフロー
{操作手順}

## データモデル
{この機能に関連するデータ構造}

## API設計
{必要なエンドポイント}

## 非機能要件
{パフォーマンス、セキュリティ等}
```

### _source_info.md

- 使用した `data_source` のタイムスタンプ
- 選択されたキーワード
- 生成時のパラメータ（モデル、温度等）
- 人間が「なぜこのアプリ案が生まれたか」を理解できる簡潔なサマリー

### CLIエントリポイント

- 実行コマンド: `pnpm generate` または `tsx scripts/generate.ts`
- 引数:
  - `--keywords`: 対象の `keyword.json` パス（省略時は最新の `data_source` 内を使用）
  - `--app-name`: 生成するアプリ名の指定（省略時はLLMが命名）
  - `--features-count`: 生成する機能数の上限（デフォルト: 5-8程度）

## 完了条件

- `pnpm generate` で `keyword.json` を元にアプリ案を生成し、`requirements/{app_name}/` 以下に所定構造で出力できる
- `overview.md` にコンセプト・機能一覧・マネタイズ・運用方針が含まれる
- 各機能の `features/*.md` に画面構成・データモデル・API設計等が含まれる
- `_source_info.md` に使用データソースの追跡情報が記録される
