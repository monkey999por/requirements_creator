# requirements_creator

各種データから連想方式でアプリケーション設計を行うためのツールです。

## ざっくりやりたいことの整理

何を作るのかと、どのような画面/機能が必要か の表面的に要件と、それぞれの機能単位の要件まで生成するツール

### 各種コマンド連携とClaudecodeによるアプリ案作成まで

このあたりはすべてコマンドラインで実行可能な単体のshell scriptや、`node`で実行可能なものとする。

1. 毎日のニュース収集を実施(cronやその他定期的なスケジューラーをアプリ外で設定)（）
    1. youtubeのランキングとかもいいね、tictokとかもありかも
    2. キーワードを抽出してベクトル化ニュースそのもののコンテキストよりも、キーワードからの連想ゲーム形式
2. アプリ作成のアイディアを作る -> 詳細な要件、マネタイズや運用方式まですべて出す。
3. ディレクトリ内に指定の構造で置く

### アプリ案作成後、それをわかりやすく見るための簡易Webページ作成

1. アプリ案を楽にみるための簡易サーバ構築(markdownのPreviewと機能ごとの階層をファーストビューで一気ににみることができる1毎のページを作る。サイドバーでどのアプリ案を見るかの切り替えが可能)
   1. サーバはNode.jsを使う

### 動作フロー

1. 起動コマンドで、NewsAPIやYoutubeのランキングAPI(暫定)を使用し、data_source/yyyy_mm_dd_hh_mm_ss/*に保存します。

### ディレクトリ構造（イメージ）

- datasource
  - yyyy_mm_dd_hh_mm_ss
    - news.json
    - {scrape_from_service}.json
    - keyword.json 　※source.jsonから生成。アプリ設計のヒントになる、重きを置くものを特に定義
- requirements
  - {app_name}
    - {使用したdata_sourceやキーワードを人間にわかる形で簡潔に整理するためのもの(ファイルでもディレクトリでも)}
    - overview.md
    - features
      - {feature_id}_{feature_name}.md
      - その他、必要なファイルやアセットがあれば配置

### サーバ側

マークダウンのpreviewができるサーバ(ブラウザから見たい)
