# 商品・サービス評価スキャン

## 概要

消費者が商品やサービスを購入・契約する際、「この選択は自分の価値観に合っているのか」を即座に判断できないという課題を解決する。バーコードスキャンまたはURL入力により、対象の商品・サービスをユーザーの価値観プロファイルと照合し、適合度スコアを即時表示する。企業の倫理性、環境負荷、価格妥当性、品質評価、サプライチェーンの透明性などを総合的に評価する。

## 画面構成

### スキャン入力画面
- カメラビュー（バーコード/QRコードスキャン用）
- テキスト入力フィールド（商品名またはURL貼り付け）
- 検索履歴の直近5件をリスト表示
- 「手動で商品を検索」リンク

### スキャン結果画面
- 商品/サービス画像とタイトル
- **総合適合スコア**（0-100、大きな円形ゲージで表示）
  - 色分け: 80以上=緑、50-79=黄、49以下=赤
- **軸別スコア内訳**（6軸の横棒グラフ）
  - 各軸にユーザーの価値観ウェイトを反映した重み付きスコア
- **評価根拠**（アコーディオン形式で展開）
  - 企業の倫理性スコアとソース
  - 環境負荷（CO2排出量、パッケージ素材等）
  - 価格比較データ
  - ユーザーレビュー集約
- 「代替を探す」CTAボタン（代替提案エンジンへ遷移）
- 「コミュニティのレビューを見る」リンク
- 「スキャン履歴に保存」ボタン

### 検索結果画面（テキスト検索時）
- 商品候補のリスト表示（画像、名称、簡易スコア）
- フィルタ（カテゴリ、価格帯）
- ページネーション

## ユーザーフロー

1. ユーザーがスキャン画面を開く
2. バーコードをカメラでスキャン、またはURLを貼り付け、またはテキストで商品名を検索する
3. システムが商品/サービスを特定し、外部APIとDBから評価データを収集する
4. ユーザーの価値観プロファイルとマッチングし、適合度スコアを算出する
5. 結果画面に総合スコアと軸別内訳が表示される
6. ユーザーが評価根拠を確認する
7. 必要に応じて「代替を探す」ボタンで代替提案エンジンに遷移する
8. スキャン結果が履歴に自動保存される

## データモデル

### products テーブル
| カラム名 | 型 | 説明 |
|----------|------|------|
| id | UUID | 主キー |
| barcode | VARCHAR(50) | バーコード（EAN/UPC） |
| name | VARCHAR(255) | 商品名 |
| brand | VARCHAR(255) | ブランド名 |
| category | VARCHAR(100) | カテゴリ |
| image_url | TEXT | 商品画像URL |
| company_id | UUID | FK: companies.id |
| external_data | JSONB | 外部APIから取得した生データ |
| embedding | vector(1536) | 商品の特徴ベクトル（pgvector） |
| updated_at | TIMESTAMP | データ更新日時 |

### companies テーブル
| カラム名 | 型 | 説明 |
|----------|------|------|
| id | UUID | 主キー |
| name | VARCHAR(255) | 企業名 |
| ethics_score | INTEGER | 倫理性スコア（0-100） |
| environment_score | INTEGER | 環境スコア（0-100） |
| labor_score | INTEGER | 労働環境スコア（0-100） |
| transparency_score | INTEGER | 透明性スコア（0-100） |
| data_sources | JSONB | スコア算出に使用したデータソース |
| updated_at | TIMESTAMP | データ更新日時 |

### scan_history テーブル
| カラム名 | 型 | 説明 |
|----------|------|------|
| id | UUID | 主キー |
| user_id | UUID | FK: users.id |
| product_id | UUID | FK: products.id |
| match_score | INTEGER | 総合適合スコア |
| score_breakdown | JSONB | 軸別スコア内訳 |
| scanned_at | TIMESTAMP | スキャン日時 |

## API設計

| メソッド | パス | 概要 |
|----------|------|------|
| POST | /api/scan/barcode | バーコードから商品を検索・評価 |
| POST | /api/scan/url | URLから商品/サービスを検索・評価 |
| GET | /api/products/search?q={query} | テキストで商品を検索 |
| GET | /api/products/:id | 商品の詳細情報と評価データを取得 |
| GET | /api/scan/history | スキャン履歴を取得 |
| DELETE | /api/scan/history/:id | 履歴を個別削除 |

## 非機能要件

- **パフォーマンス**: バーコードスキャン後、結果表示まで3秒以内。キャッシュ済み商品は1秒以内。商品データはRedisに24時間キャッシュ
- **精度**: バーコード認識精度99%以上（明るさ不足時はフラッシュ案内を表示）
- **レート制限**: 無料プランは月10回、プレミアムプランは無制限。レート制限はUpstash Redisで管理
- **データ鮮度**: 企業評価データは日次バッチ更新。ユーザーフィードバックによるスコア補正は即時反映
- **セキュリティ**: スキャン履歴は本人のみ閲覧可能。カメラ権限はスキャン時のみ使用し、画像データはサーバーに送信しない
- **対応フォーマット**: EAN-13、EAN-8、UPC-A、QRコード
