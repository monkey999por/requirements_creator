# 代替提案エンジン

## 概要

ユーザーが現在利用中のサービスや購入予定の商品が自分の価値観と合っていない場合に、「では何に切り替えればよいのか」がわからないという課題を解決する。価値観プロファイルと商品/サービスのベクトルデータを活用し、同カテゴリ内でより価値観適合度の高い代替候補をAIがランキング形式で提案する。単なる類似品検索ではなく、ユーザー固有の価値観に最適化された提案を行う。

## 画面構成

### 代替提案画面
- 現在の商品/サービスのカード（名称、適合スコア）
- 「この商品の代替を探す理由」（どの価値観軸でスコアが低いかを表示）
- **代替候補リスト**（最大10件、スコア降順）
  - 各候補カード:
    - 商品/サービス画像、名称、ブランド
    - 適合スコア（現在の商品との差分を+/-表示）
    - 軸別スコア比較（ミニバーチャート）
    - 価格（差分表示）
    - 「詳細を見る」ボタン
    - 「購入リンク」ボタン（外部サイトへ遷移）
- フィルタ:
  - 価格帯（スライダー）
  - 入手性（オンライン/実店舗/両方）
  - 特定軸の重視度調整

### 一括代替診断画面
- ユーザーが利用中のサービス一覧（手動登録またはサブスク連携）
- 各サービスの適合スコアと「代替を探す」ボタン
- 「全サービスを一括診断」CTAボタン
- 切り替え推奨度が高い順にソートされたサマリー

## ユーザーフロー

1. ユーザーが商品スキャン結果画面から「代替を探す」をタップ、または一括代替診断画面を開く
2. システムが元商品と同カテゴリの商品をベクトル検索で取得する
3. 各候補に対してユーザーの価値観プロファイルとの適合度を算出する
4. 適合度が高い順に代替候補リストを表示する
5. ユーザーがフィルタで絞り込みや軸の重み調整を行う
6. 気になる候補の「詳細を見る」で評価根拠を確認する
7. 「購入リンク」で外部サイトに遷移し購入/契約する
8. 切り替え履歴が記録され、月次レポートに反映される

## データモデル

### alternatives_cache テーブル
| カラム名 | 型 | 説明 |
|----------|------|------|
| id | UUID | 主キー |
| source_product_id | UUID | FK: products.id（元の商品） |
| alternative_product_id | UUID | FK: products.id（代替商品） |
| category_similarity | FLOAT | カテゴリ類似度 |
| computed_at | TIMESTAMP | 算出日時 |

### user_services テーブル
| カラム名 | 型 | 説明 |
|----------|------|------|
| id | UUID | 主キー |
| user_id | UUID | FK: users.id |
| service_name | VARCHAR(255) | サービス名 |
| category | VARCHAR(100) | カテゴリ |
| monthly_cost | INTEGER | 月額費用（円） |
| product_id | UUID | FK: products.id（商品DBとの紐付け） |
| registered_at | TIMESTAMP | 登録日時 |

### switch_history テーブル
| カラム名 | 型 | 説明 |
|----------|------|------|
| id | UUID | 主キー |
| user_id | UUID | FK: users.id |
| from_product_id | UUID | FK: products.id |
| to_product_id | UUID | FK: products.id |
| score_improvement | INTEGER | スコア向上幅 |
| switched_at | TIMESTAMP | 切り替え日時 |

## API設計

| メソッド | パス | 概要 |
|----------|------|------|
| GET | /api/alternatives/:productId | 指定商品の代替候補を取得（価値観スコア付き） |
| POST | /api/alternatives/bulk-diagnose | 登録済みサービス一覧の一括診断 |
| GET | /api/user-services | ユーザーの登録サービス一覧を取得 |
| POST | /api/user-services | サービスを手動登録 |
| DELETE | /api/user-services/:id | 登録サービスを削除 |
| POST | /api/switch-history | 切り替え履歴を記録 |
| GET | /api/switch-history | 切り替え履歴を取得 |

## 非機能要件

- **パフォーマンス**: 代替候補の検索・スコアリングは5秒以内に完了。pgvectorの近傍検索を活用し、候補取得自体は1秒以内
- **提案品質**: 代替候補は同カテゴリかつ入手可能な商品のみ。ユーザーフィードバック（「この提案は役立った/役立たなかった」）を収集し、提案精度を継続改善
- **透明性**: なぜその代替が推奨されるのかの根拠を必ず表示（「環境スコアが+30高い」「地域の店舗で購入可能」等）
- **アフィリエイトの透明性**: 収益を得ている推薦にはアフィリエイトマークを明示。スコアリングにはアフィリエイト有無を反映させない
- **データ鮮度**: 代替候補キャッシュは7日間有効。価格データは日次更新
