# 価値観プロファイリング

## 概要

ユーザーが「自分がどんな価値観を持っているか」を正確に把握できていないという課題を解決する。対話形式の質問とシナリオ選択を通じて、倫理性・環境配慮・コスト重視・品質重視・地域経済貢献・プライバシー重視の6軸で価値観を数値化し、レーダーチャートで可視化する。このプロファイルが他の全機能（商品評価・代替提案・投資診断等）のスコアリング基盤となる。

## 画面構成

### オンボーディング画面（初回のみ）
- アプリのコンセプト説明カード（3枚スワイプ）
- 「価値観診断を始める」CTAボタン

### 診断画面
- プログレスバー（全15問中の進捗表示）
- 質問カード: シナリオベースの2択〜4択形式
  - 例: 「同じ商品がAmazonで500円安く買えます。地元の商店で買いますか？」
  - 各選択肢にアイコンとテキスト
- 「前の質問に戻る」ボタン
- スキップリンク（任意回答制）

### 結果画面
- 6軸レーダーチャート（Rechartsで描画）
  - 倫理性 / 環境配慮 / コスト重視 / 品質重視 / 地域経済貢献 / プライバシー重視
- 各軸のスコア（0-100）と簡潔な説明テキスト
- 「あなたの価値観タイプ」ラベル（例: 「エシカルパイオニア」「バランス重視の合理主義者」）
- 「プロファイルを保存する」CTAボタン
- 「再診断する」リンク

### 設定画面（プロファイル編集）
- 各軸のスライダーによる手動調整
- 診断履歴の一覧表示
- 「再診断する」ボタン

## ユーザーフロー

1. ユーザーがアプリに初回ログインする
2. オンボーディング画面でアプリのコンセプトを確認する
3. 「価値観診断を始める」をタップする
4. シナリオベースの質問に順番に回答する（全15問、所要3〜5分）
5. システムが回答を分析し、6軸の価値観スコアを算出する
6. レーダーチャートと価値観タイプが結果画面に表示される
7. ユーザーが結果を確認し、必要に応じてスライダーで微調整する
8. 「プロファイルを保存する」をタップして完了
9. 保存されたプロファイルが他機能のスコアリング基盤として利用される

## データモデル

### users テーブル
| カラム名 | 型 | 説明 |
|----------|------|------|
| id | UUID | 主キー |
| email | VARCHAR(255) | メールアドレス |
| name | VARCHAR(100) | 表示名 |
| auth_provider | VARCHAR(50) | 認証プロバイダ（google/apple/email） |
| created_at | TIMESTAMP | 作成日時 |
| updated_at | TIMESTAMP | 更新日時 |

### value_profiles テーブル
| カラム名 | 型 | 説明 |
|----------|------|------|
| id | UUID | 主キー |
| user_id | UUID | FK: users.id |
| ethics_score | INTEGER | 倫理性スコア（0-100） |
| environment_score | INTEGER | 環境配慮スコア（0-100） |
| cost_score | INTEGER | コスト重視スコア（0-100） |
| quality_score | INTEGER | 品質重視スコア（0-100） |
| local_economy_score | INTEGER | 地域経済貢献スコア（0-100） |
| privacy_score | INTEGER | プライバシー重視スコア（0-100） |
| value_type | VARCHAR(50) | 価値観タイプラベル |
| is_active | BOOLEAN | 現在有効なプロファイルか |
| created_at | TIMESTAMP | 作成日時 |

### diagnosis_sessions テーブル
| カラム名 | 型 | 説明 |
|----------|------|------|
| id | UUID | 主キー |
| user_id | UUID | FK: users.id |
| profile_id | UUID | FK: value_profiles.id |
| answers | JSONB | 回答データ（質問ID:選択肢IDの配列） |
| completed_at | TIMESTAMP | 完了日時 |

## API設計

| メソッド | パス | 概要 |
|----------|------|------|
| GET | /api/diagnosis/questions | 診断質問一覧を取得 |
| POST | /api/diagnosis/submit | 診断回答を送信しプロファイルを生成 |
| GET | /api/profiles/me | 現在のユーザーのアクティブプロファイルを取得 |
| PUT | /api/profiles/:id | プロファイルのスコアを手動調整 |
| GET | /api/profiles/history | 診断履歴一覧を取得 |
| POST | /api/profiles/:id/activate | 指定プロファイルをアクティブに設定 |

## 非機能要件

- **パフォーマンス**: 診断結果の算出は2秒以内に完了すること。レーダーチャートのレンダリングは60fpsを維持
- **アクセシビリティ**: スクリーンリーダー対応。色覚多様性に配慮したチャート配色。キーボードナビゲーション対応
- **セキュリティ**: 価値観データは個人の思想信条に関わる機密情報のため、AES-256で暗号化保存。API認証にJWTを使用
- **データ保持**: ユーザーがアカウント削除時にはプロファイルデータも完全削除（GDPR準拠）
- **オフライン対応**: 診断質問はキャッシュしオフラインでも回答可能。送信はオンライン復帰時に実行
