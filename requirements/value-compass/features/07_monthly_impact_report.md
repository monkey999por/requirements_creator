# 月次インパクトレポート

## 概要

エシカル消費や価値観に基づいた選択を実践しても、「自分の行動がどれだけ一貫しているか」「どの程度改善できたか」を客観的に振り返る手段がないという課題を解決する。1ヶ月間のスキャン履歴・代替切り替え・投資判断・メディア消費パターンを集計し、価値観プロファイルに対する行動の一致度をレポートとして可視化する。成果をポジティブに振り返ることで、持続的な行動変容を促す。

## 画面構成

### レポートトップ画面
- 対象月の選択（月別タブ）
- **総合スコアカード**
  - 月間価値観一致度（0-100のゲージ）
  - 前月比の変化（+/-表示）
  - 「今月のハイライト」テキスト（AIが生成する1文サマリー）

### レポート詳細画面（スクロール形式の1ページレポート）
- **セクション1: スキャン活動サマリー**
  - 月間スキャン回数
  - スキャン商品の平均適合スコア
  - スコア分布ヒストグラム
  - 「最も価値観に合った商品」「最も合わなかった商品」のピックアップ

- **セクション2: 代替切り替え実績**
  - 切り替え件数
  - 切り替えによる平均スコア向上幅
  - コスト変化（月額の増減）
  - 切り替え履歴のタイムライン表示

- **セクション3: 投資ポートフォリオの変化**（プレミアムプラン）
  - 月初と月末のポートフォリオ価値観スコア比較
  - 改善された銘柄・悪化した銘柄のハイライト

- **セクション4: メディア消費傾向**（プレミアムプラン）
  - 分析した記事数
  - 閲覧メディアのバイアス傾向分布
  - 「バランスの取れた情報収集度」スコア

- **セクション5: 来月へのアドバイス**
  - AIが生成する改善提案（具体的なアクション3つ）
  - 「来月チャレンジする価値観軸」の提案

### レポート共有画面
- レポートのサマリーカード画像（SNS共有用）
- 共有ボタン（X/Instagram/LINE）
- 共有内容のプレビュー

## ユーザーフロー

1. 月初（毎月1日）にプッシュ通知で「先月のレポートが完成しました」と通知される
2. ユーザーがレポートトップ画面を開く
3. 総合スコアとハイライトを確認する
4. 詳細セクションをスクロールして各項目の実績を振り返る
5. 来月へのアドバイスを確認し、改善目標を意識する
6. 必要に応じてレポートサマリーをSNSに共有する
7. 過去の月を選択して推移を確認する

## データモデル

### monthly_reports テーブル
| カラム名 | 型 | 説明 |
|----------|------|------|
| id | UUID | 主キー |
| user_id | UUID | FK: users.id |
| year_month | VARCHAR(7) | 対象年月（例: "2026-02"） |
| overall_score | INTEGER | 月間総合スコア（0-100） |
| previous_score | INTEGER | 前月スコア（nullable） |
| highlight_text | TEXT | AIが生成したハイライトテキスト |
| generated_at | TIMESTAMP | レポート生成日時 |

### report_sections テーブル
| カラム名 | 型 | 説明 |
|----------|------|------|
| id | UUID | 主キー |
| report_id | UUID | FK: monthly_reports.id |
| section_type | VARCHAR(50) | セクション種別（scan/switch/portfolio/media/advice） |
| data | JSONB | セクション固有の集計データ |
| sort_order | INTEGER | 表示順 |

### report_shares テーブル
| カラム名 | 型 | 説明 |
|----------|------|------|
| id | UUID | 主キー |
| report_id | UUID | FK: monthly_reports.id |
| platform | VARCHAR(50) | 共有先プラットフォーム |
| share_image_url | TEXT | 共有用画像URL |
| shared_at | TIMESTAMP | 共有日時 |

## API設計

| メソッド | パス | 概要 |
|----------|------|------|
| GET | /api/reports | 月次レポート一覧を取得 |
| GET | /api/reports/:yearMonth | 指定月のレポート詳細を取得 |
| POST | /api/reports/:yearMonth/generate | レポートを手動生成（通常は自動生成） |
| GET | /api/reports/:yearMonth/share-image | SNS共有用のサマリー画像を取得 |
| POST | /api/reports/:yearMonth/share | 共有履歴を記録 |
| GET | /api/reports/trend | 月次スコア推移データを取得 |

## 非機能要件

- **レポート生成**: 月次レポートは毎月1日の早朝にバッチジョブで全ユーザー分を生成。1ユーザーあたり30秒以内に完了
- **パフォーマンス**: レポート画面の表示は2秒以内。チャート類のレンダリングはクライアントサイドで実行
- **通知**: プッシュ通知はFirebase Cloud Messagingを使用。通知タイミングはユーザーのタイムゾーンに合わせて朝9時に配信
- **共有画像**: OGP対応の画像を動的生成（Satori/resvg-js使用）。各SNSの推奨画像サイズに対応
- **データ保持**: レポートデータは無期限保存。ユーザーは過去の全レポートを閲覧可能
- **プライバシー**: SNS共有時は総合スコアと価値観タイプのみ表示。具体的な商品名やポートフォリオ情報は含まない
