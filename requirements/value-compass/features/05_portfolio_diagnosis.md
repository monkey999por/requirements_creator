# 投資ポートフォリオ価値観診断

## 概要

ESG投資や倫理的投資に関心があるものの、「自分のポートフォリオが実際に自分の価値観に合っているのか」を評価する手段がないという課題を解決する。ユーザーの保有資産や投資先企業を価値観軸（倫理性・環境負荷・社会貢献度・ガバナンス・労働環境）で評価し、ポートフォリオ全体の価値観適合スコアを算出。スコアが低い銘柄に対しては価値観により合致する代替銘柄を提案する。

## 画面構成

### ポートフォリオ入力画面
- 銘柄追加フォーム（ティッカーシンボルまたは企業名で検索）
- 保有数量・取得単価入力フィールド
- 「CSVインポート」ボタン（証券会社のエクスポートデータに対応）
- 登録済み銘柄リスト（編集・削除可能）

### ポートフォリオ診断結果画面
- **ポートフォリオ全体の価値観適合スコア**（大きな円形ゲージ、0-100）
- **価値観軸別のポートフォリオスコア**（レーダーチャート）
  - ユーザーの価値観プロファイルと重ね合わせ表示
- **銘柄別スコアランキング**（テーブル形式）
  - 銘柄名、ティッカー、保有比率、価値観適合スコア、ESGスコア
  - スコアが低い銘柄にはアラートアイコン表示
  - 各行に「代替を探す」ボタン
- **改善提案セクション**
  - 「この銘柄を入れ替えるとポートフォリオスコアが+XX向上します」形式の具体的提案
  - 提案される代替銘柄のリスク・リターン比較

### 銘柄詳細画面
- 企業の価値観軸別スコア（6軸棒グラフ）
- ESGデータサマリー（CO2排出量、ダイバーシティ指標、ガバナンス構造等）
- 直近のニュース・スキャンダル情報
- 「この企業の代替投資先」リスト

## ユーザーフロー

1. ユーザーがポートフォリオ入力画面で保有銘柄を登録する（手動入力またはCSVインポート）
2. 「診断する」ボタンをタップする
3. システムが各銘柄の企業データ・ESGデータを外部APIから取得する
4. ユーザーの価値観プロファイルとマッチングし、銘柄別・ポートフォリオ全体のスコアを算出する
5. 診断結果画面にスコアとランキングが表示される
6. ユーザーがスコアの低い銘柄を確認し、「代替を探す」で代替銘柄を検討する
7. 改善提案を参考にポートフォリオの組み替えを検討する
8. 診断結果が保存され、月次レポートで推移を追跡できる

## データモデル

### portfolios テーブル
| カラム名 | 型 | 説明 |
|----------|------|------|
| id | UUID | 主キー |
| user_id | UUID | FK: users.id |
| name | VARCHAR(100) | ポートフォリオ名（デフォルト: "メイン"） |
| created_at | TIMESTAMP | 作成日時 |
| updated_at | TIMESTAMP | 更新日時 |

### portfolio_holdings テーブル
| カラム名 | 型 | 説明 |
|----------|------|------|
| id | UUID | 主キー |
| portfolio_id | UUID | FK: portfolios.id |
| ticker | VARCHAR(20) | ティッカーシンボル |
| company_name | VARCHAR(255) | 企業名 |
| shares | DECIMAL(12,4) | 保有数量 |
| cost_basis | DECIMAL(12,2) | 取得単価 |
| added_at | TIMESTAMP | 追加日時 |

### company_esg_scores テーブル
| カラム名 | 型 | 説明 |
|----------|------|------|
| id | UUID | 主キー |
| ticker | VARCHAR(20) | ティッカーシンボル |
| company_name | VARCHAR(255) | 企業名 |
| ethics_score | INTEGER | 倫理性スコア（0-100） |
| environment_score | INTEGER | 環境スコア（0-100） |
| social_score | INTEGER | 社会貢献スコア（0-100） |
| governance_score | INTEGER | ガバナンススコア（0-100） |
| labor_score | INTEGER | 労働環境スコア（0-100） |
| controversy_flags | JSONB | 直近のスキャンダル・論争フラグ |
| data_sources | JSONB | スコア算出元データソース |
| updated_at | TIMESTAMP | データ更新日時 |

### portfolio_diagnoses テーブル
| カラム名 | 型 | 説明 |
|----------|------|------|
| id | UUID | 主キー |
| portfolio_id | UUID | FK: portfolios.id |
| overall_score | INTEGER | ポートフォリオ全体の適合スコア |
| axis_scores | JSONB | 軸別スコア |
| holdings_scores | JSONB | 銘柄別スコア詳細 |
| suggestions | JSONB | 改善提案データ |
| diagnosed_at | TIMESTAMP | 診断日時 |

## API設計

| メソッド | パス | 概要 |
|----------|------|------|
| GET | /api/portfolios | ユーザーのポートフォリオ一覧を取得 |
| POST | /api/portfolios | 新規ポートフォリオを作成 |
| POST | /api/portfolios/:id/holdings | 銘柄を追加 |
| POST | /api/portfolios/:id/import | CSVから銘柄を一括インポート |
| DELETE | /api/portfolios/:id/holdings/:holdingId | 銘柄を削除 |
| POST | /api/portfolios/:id/diagnose | ポートフォリオ診断を実行 |
| GET | /api/portfolios/:id/diagnoses | 診断履歴を取得 |
| GET | /api/companies/:ticker/esg | 企業のESG・価値観スコアを取得 |
| GET | /api/companies/:ticker/alternatives | 代替銘柄候補を取得 |

## 非機能要件

- **パフォーマンス**: 20銘柄までのポートフォリオ診断は10秒以内に完了。企業ESGデータは6時間キャッシュ
- **データ精度**: ESGスコアは複数のデータソース（MSCI ESG、Sustainalytics等の公開データ）を組み合わせて算出。スコア算出根拠を明示
- **免責事項**: 投資助言には該当しない旨の免責表示を全画面に配置。「この情報は投資判断の参考情報であり、投資助言ではありません」を明記
- **対応市場**: 初期リリースでは日本株（東証）と米国株（NYSE/NASDAQ）に対応
- **セキュリティ**: ポートフォリオ情報は個人資産に関する機密データのため、暗号化保存。APIアクセスは本人認証必須
- **プライバシー**: 個人の投資データは統計目的であっても第三者に提供しない
