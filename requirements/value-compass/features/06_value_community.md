# 価値観コミュニティ

## 概要

エシカル消費や代替サービスへの切り替えを実践する際、「本当に使い勝手は良いのか」「他の人はどう感じているか」という実体験ベースの情報が不足しているという課題を解決する。近い価値観を持つユーザー同士が代替商品・サービスのレビューや体験談を共有し、互いの意思決定を支援するコミュニティ機能を提供する。価値観プロファイルの類似度に基づいたマッチングにより、自分にとって最も参考になるレビューが優先表示される。

## 画面構成

### コミュニティトップ画面
- **おすすめレビュー**（価値観が近いユーザーのレビューを優先表示）
- **トレンド投稿**（直近で注目されている切り替え事例）
- カテゴリタブ（食品/テック/金融/メディア/日用品/その他）
- 検索バー（商品名・サービス名・キーワード）
- 「レビューを書く」FABボタン

### レビュー投稿画面
- 対象商品/サービスの選択（商品DBからの検索、またはフリーテキスト入力）
- 「何からの切り替えか」（切り替え元の商品/サービスを選択）
- 星評価（5段階）
- テキストレビュー（最大1000文字）
- 画像添付（最大3枚）
- タグ選択（コスパ良し/環境にやさしい/使いやすい/初心者向け 等）
- 公開範囲設定（全体公開/価値観が近い人のみ/非公開）

### レビュー詳細画面
- 投稿者のプロフィール（価値観タイプラベル、価値観類似度%）
- レビュー本文と画像
- 切り替え前後の比較情報（スコア差、コスト差）
- 「参考になった」ボタン
- コメント欄（スレッド形式）
- 「この商品をスキャンする」リンク

### ユーザープロフィール画面
- 価値観タイプラベルとレーダーチャート（詳細スコアは非公開）
- 投稿したレビュー一覧
- 「参考になった」を受けた数
- フォロー/フォロワー数

## ユーザーフロー

1. ユーザーがコミュニティトップ画面を開く
2. 価値観類似度に基づいて優先表示されたレビューを閲覧する
3. 気になる商品/サービスのレビュー詳細を確認する
4. 「参考になった」ボタンで有用なレビューを評価する
5. 自分もレビューを書きたい場合は「レビューを書く」をタップする
6. 対象商品、切り替え元、星評価、テキスト、画像を入力して投稿する
7. 投稿されたレビューが価値観の近いユーザーのフィードに表示される
8. コメント欄で質問・回答のやり取りを行う

## データモデル

### reviews テーブル
| カラム名 | 型 | 説明 |
|----------|------|------|
| id | UUID | 主キー |
| user_id | UUID | FK: users.id |
| product_id | UUID | FK: products.id（対象商品） |
| switched_from_id | UUID | FK: products.id（切り替え元、nullable） |
| rating | INTEGER | 星評価（1-5） |
| body | TEXT | レビュー本文 |
| tags | VARCHAR(50)[] | タグ配列 |
| visibility | VARCHAR(20) | 公開範囲（public/similar_values/private） |
| helpful_count | INTEGER | 「参考になった」数 |
| created_at | TIMESTAMP | 作成日時 |
| updated_at | TIMESTAMP | 更新日時 |

### review_images テーブル
| カラム名 | 型 | 説明 |
|----------|------|------|
| id | UUID | 主キー |
| review_id | UUID | FK: reviews.id |
| image_url | TEXT | 画像URL |
| sort_order | INTEGER | 表示順 |

### review_comments テーブル
| カラム名 | 型 | 説明 |
|----------|------|------|
| id | UUID | 主キー |
| review_id | UUID | FK: reviews.id |
| user_id | UUID | FK: users.id |
| parent_comment_id | UUID | FK: review_comments.id（スレッド用、nullable） |
| body | TEXT | コメント本文 |
| created_at | TIMESTAMP | 作成日時 |

### review_helpful テーブル
| カラム名 | 型 | 説明 |
|----------|------|------|
| id | UUID | 主キー |
| review_id | UUID | FK: reviews.id |
| user_id | UUID | FK: users.id |
| created_at | TIMESTAMP | 作成日時 |

### user_follows テーブル
| カラム名 | 型 | 説明 |
|----------|------|------|
| id | UUID | 主キー |
| follower_id | UUID | FK: users.id（フォローする側） |
| following_id | UUID | FK: users.id（フォローされる側） |
| created_at | TIMESTAMP | 作成日時 |

## API設計

| メソッド | パス | 概要 |
|----------|------|------|
| GET | /api/community/feed | パーソナライズされたレビューフィードを取得 |
| GET | /api/community/trending | トレンドのレビューを取得 |
| POST | /api/reviews | レビューを投稿 |
| GET | /api/reviews/:id | レビュー詳細を取得 |
| PUT | /api/reviews/:id | レビューを編集 |
| DELETE | /api/reviews/:id | レビューを削除 |
| POST | /api/reviews/:id/helpful | 「参考になった」を記録 |
| POST | /api/reviews/:id/comments | コメントを投稿 |
| GET | /api/reviews/:id/comments | コメント一覧を取得 |
| POST | /api/users/:id/follow | ユーザーをフォロー |
| DELETE | /api/users/:id/follow | フォロー解除 |
| GET | /api/users/:id/profile | ユーザープロフィールを取得 |

## 非機能要件

- **パフォーマンス**: フィードの初期表示は2秒以内。無限スクロールで追加読み込みは1秒以内
- **スケーラビリティ**: フィードの生成はバックグラウンドジョブで事前計算。価値観類似度の計算はpgvectorのコサイン類似度を使用
- **コンテンツモデレーション**: 投稿内容はAIによる自動モデレーション（スパム、ヘイトスピーチ、虚偽情報の検出）。報告機能による手動レビュー
- **プライバシー**: 価値観プロファイルの詳細スコアは非公開。価値観タイプラベルと類似度%のみ他ユーザーに表示
- **画像**: アップロード画像は最大5MB/枚。WebP形式に自動変換。SupabaseのStorageに保存
- **不正行為対策**: 同一ユーザーによる同一商品への複数レビュー禁止。「参考になった」の自作自演検出
