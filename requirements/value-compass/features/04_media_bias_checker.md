# メディアバイアスチェッカー

## 概要

ニュースを読む際に「この記事は偏っていないか」「他のメディアではどう報じているか」を個人が確認するのは多大な時間と労力を要するという課題を解決する。ニュース記事のURLを入力すると、同一トピックを報じる複数メディアの記事を自動収集し、論調・視点の違いをAIが分析。バイアスの方向性と度合いを可視化することで、ユーザーのメディアリテラシー向上を支援する。

## 画面構成

### 記事入力画面
- URL入力フィールド（ペースト用の大きなテキストエリア）
- 「分析する」CTAボタン
- 最近の分析履歴リスト（直近5件）
- トレンドトピック（現在話題のニューストピックから選択可能）

### 分析結果画面
- **入力記事のサマリー**（タイトル、メディア名、要約）
- **バイアスメーター**（左寄り←→右寄りのスペクトラム上に配置）
  - 対象記事のポジション表示
- **同一トピック報道の比較テーブル**
  - 各メディアの記事タイトル、論調の方向性、感情トーン（ポジティブ/ネガティブ/中立）
  - 最大8メディアまで比較
- **論点マップ**（ツリーマップまたはサンキーダイアグラム）
  - 各メディアがどの論点を重視しているかを視覚化
- **AIによるバイアス解説**
  - 使用されている修辞技法、省略されている情報、フレーミングの分析
- **「バランスの取れた理解のために」推薦記事リスト**
  - 異なる視点の記事を2〜3件推薦
- 「コミュニティの意見を見る」リンク

### 分析履歴画面
- 過去の分析結果一覧（日付、記事タイトル、バイアス度合い）
- フィルタ（期間、トピック）
- 「月間メディア消費傾向」サマリーカード

## ユーザーフロー

1. ユーザーがニュース記事のURLを入力する、またはトレンドトピックから選択する
2. システムが記事内容を取得・解析する
3. NewsAPI等で同一トピックの他メディア記事を自動収集する
4. AIが各記事の論調・バイアス方向性・感情トーン・フレーミングを分析する
5. 分析結果画面にバイアスメーター、比較テーブル、論点マップが表示される
6. ユーザーがAIのバイアス解説を確認する
7. 推薦された異なる視点の記事を読み、多角的な理解を得る
8. 分析結果が履歴に保存され、月次レポートに反映される

## データモデル

### media_sources テーブル
| カラム名 | 型 | 説明 |
|----------|------|------|
| id | UUID | 主キー |
| name | VARCHAR(255) | メディア名 |
| domain | VARCHAR(255) | ドメイン |
| bias_tendency | VARCHAR(50) | 一般的なバイアス傾向 |
| reliability_score | INTEGER | 信頼性スコア（0-100） |
| updated_at | TIMESTAMP | データ更新日時 |

### bias_analyses テーブル
| カラム名 | 型 | 説明 |
|----------|------|------|
| id | UUID | 主キー |
| user_id | UUID | FK: users.id |
| source_url | TEXT | 分析対象の記事URL |
| source_media_id | UUID | FK: media_sources.id |
| topic_summary | TEXT | トピック要約 |
| bias_score | INTEGER | バイアス度合い（-100:左寄り〜+100:右寄り） |
| sentiment | VARCHAR(20) | 感情トーン（positive/negative/neutral） |
| ai_analysis | JSONB | AI分析結果（修辞技法、フレーミング等） |
| analyzed_at | TIMESTAMP | 分析日時 |

### compared_articles テーブル
| カラム名 | 型 | 説明 |
|----------|------|------|
| id | UUID | 主キー |
| analysis_id | UUID | FK: bias_analyses.id |
| media_id | UUID | FK: media_sources.id |
| article_url | TEXT | 記事URL |
| title | VARCHAR(500) | 記事タイトル |
| bias_score | INTEGER | バイアス度合い |
| sentiment | VARCHAR(20) | 感情トーン |
| key_points | JSONB | 主要論点 |

## API設計

| メソッド | パス | 概要 |
|----------|------|------|
| POST | /api/media/analyze | 記事URLを送信してバイアス分析を実行 |
| GET | /api/media/analyses | 分析履歴を取得 |
| GET | /api/media/analyses/:id | 特定の分析結果詳細を取得 |
| GET | /api/media/trending | トレンドトピック一覧を取得 |
| POST | /api/media/analyze-topic | トピックベースで複数メディアの比較分析を実行 |
| GET | /api/media/sources | メディアソース一覧と信頼性スコアを取得 |

## 非機能要件

- **パフォーマンス**: 記事分析は10秒以内に完了（同一トピック記事の収集・AI分析含む）。キャッシュ済みトピックは3秒以内
- **精度**: バイアス判定はAIモデルの分析結果に加え、メディア自体の既知バイアスデータを組み合わせて判定。判定根拠を必ず明示
- **中立性**: アプリ自体が特定の政治的立場を取らないよう、「バイアスがある=悪い」ではなく「どの方向にどの程度偏りがあるか」の情報提供に徹する
- **対応言語**: 初期リリースでは日本語・英語のニュース記事に対応
- **レート制限**: プレミアムプラン限定機能。月間100回の分析まで（NewsAPIの制限に準拠）
- **セキュリティ**: 分析対象URLは本人のみ閲覧可。ユーザーの閲覧傾向データは匿名化して統計のみに利用
