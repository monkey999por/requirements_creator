# サービス連携・利用状況スキャン

## 概要

ユーザーが日常的に利用しているデジタルサービスを把握することが、エシカル度の可視化の第一歩となる。しかし、自分が利用しているサービスを正確に把握している人は少ない。本機能は、OAuth連携やメール受信履歴の分析により利用サービスを自動検出するとともに、手動登録にも対応し、ユーザーのデジタルライフの全体像を構築する。

## 画面構成

### サービス一覧画面
- **ヘッダー**: 登録済みサービス数、自動検出候補の通知バッジ
- **サービスカードリスト**: 各サービスのアイコン・名称・カテゴリ・エシカルスコア（バッジ表示）を一覧表示。カテゴリ（SNS/EC/サブスク/クラウド等）でフィルタリング可能
- **自動検出候補セクション**: OAuth連携やメール解析で検出されたサービスを「追加候補」として提示。ワンタップで登録リストに追加
- **手動追加ボタン**: サービス名を検索して手動登録するモーダルを起動

### サービス連携設定画面
- **OAuth連携リスト**: Google、Apple、Microsoft等のアカウント連携状態を表示。未連携のアカウントには「連携する」ボタン
- **メール解析設定**: メール受信履歴からのサービス検出を許可するトグル。解析対象期間の設定
- **プライバシー説明セクション**: どのデータをどの目的で使用するかの透明な説明

## ユーザーフロー

1. ユーザーが初回セットアップでOAuthアカウント連携を選択する
2. システムがOAuth認可フローを実行し、アカウント情報から利用サービスを自動検出する
3. システムが検出されたサービスを候補リストとして表示する
4. ユーザーが候補リストから追加するサービスを選択・確認する
5. ユーザーが必要に応じてサービス検索から手動でサービスを追加する
6. システムが登録されたサービスのエシカルスコアを取得・表示する
7. 以降、定期的にバックグラウンドで新規サービス利用を検出し、候補として通知する

## データモデル

### users テーブル
| カラム名 | 型 | 説明 |
|----------|------|------|
| id | UUID | 主キー |
| email | VARCHAR(255) | メールアドレス |
| display_name | VARCHAR(100) | 表示名 |
| created_at | TIMESTAMP | 作成日時 |
| updated_at | TIMESTAMP | 更新日時 |

### user_services テーブル
| カラム名 | 型 | 説明 |
|----------|------|------|
| id | UUID | 主キー |
| user_id | UUID | users.id への外部キー |
| service_id | UUID | services.id への外部キー |
| detection_method | ENUM('oauth', 'email', 'manual') | 検出方法 |
| is_active | BOOLEAN | 現在利用中か |
| added_at | TIMESTAMP | 登録日時 |

### services テーブル
| カラム名 | 型 | 説明 |
|----------|------|------|
| id | UUID | 主キー |
| name | VARCHAR(200) | サービス名 |
| company_id | UUID | companies.id への外部キー |
| category | VARCHAR(50) | カテゴリ（SNS/EC/サブスク等） |
| icon_url | TEXT | アイコン画像URL |
| website_url | TEXT | 公式サイトURL |

### oauth_connections テーブル
| カラム名 | 型 | 説明 |
|----------|------|------|
| id | UUID | 主キー |
| user_id | UUID | users.id への外部キー |
| provider | VARCHAR(50) | プロバイダ名（google, apple等） |
| access_token_encrypted | TEXT | 暗号化アクセストークン |
| refresh_token_encrypted | TEXT | 暗号化リフレッシュトークン |
| scopes | TEXT[] | 許可スコープ |
| connected_at | TIMESTAMP | 連携日時 |

## API設計

| メソッド | パス | 概要 |
|----------|------|------|
| GET | /api/services/me | ユーザーの登録済みサービス一覧を取得 |
| POST | /api/services/me | サービスを手動登録 |
| DELETE | /api/services/me/:serviceId | 登録済みサービスを削除 |
| GET | /api/services/search | サービス名で検索（手動追加用） |
| GET | /api/services/detect | OAuth連携から利用サービスを自動検出 |
| POST | /api/oauth/connect | OAuthアカウント連携を開始 |
| DELETE | /api/oauth/connect/:provider | OAuth連携を解除 |
| GET | /api/oauth/status | 現在のOAuth連携状態を取得 |

## 非機能要件

- **パフォーマンス**: サービス一覧の表示は500ms以内。自動検出処理はバックグラウンドで実行し、完了時にプッシュ通知
- **セキュリティ**: OAuthトークンはAES-256-GCMで暗号化保存。トークンのスコープは必要最小限（読み取り専用）に制限。メール解析はサーバーサイドで実行し、メール本文は保存しない（サービス名のみ抽出）
- **プライバシー**: ユーザーはいつでもOAuth連携を解除可能。連携解除時は関連データを即座に削除。どのデータを取得するか事前に明示
- **アクセシビリティ**: サービスカードはスクリーンリーダー対応。キーボードナビゲーションでの操作を保証
