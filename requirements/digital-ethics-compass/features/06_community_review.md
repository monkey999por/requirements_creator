# コミュニティレビュー

## 概要

エシカルなサービス選択において、実際の乗り換え体験者の声は非常に参考になる。公式情報やAIスコアだけでは分からない「実際に使ってみた感想」「乗り換えで困ったこと」「想定外のメリット」といったリアルな情報を、ユーザー同士が共有するコミュニティ機能を提供する。スコアリングの透明性を高め、ユーザー参加型のエシカル評価エコシステムを構築する。

## 画面構成

### レビューフィード画面
- **カテゴリタブ**: 「乗り換え体験記」「サービスレビュー」「ディスカッション」のタブ切り替え
- **レビューカードリスト**: 投稿者アバター（匿名可）、対象サービス名、評価星（1-5）、タイトル、冒頭テキストのプレビュー
- **並び替え**: 新着順、参考になった数順、対象サービス別
- **投稿ボタン**: 新規レビュー投稿画面へ遷移するFAB

### レビュー投稿画面
- **レビュータイプ選択**: 乗り換え体験記/サービスレビュー/ディスカッション
- **対象サービス選択**: サービスをサジェスト付き検索で選択（乗り換え体験記の場合は元サービスと移行先の両方）
- **評価入力**: 星評価（1-5）+ 軸別の満足度（任意）
- **テキスト入力**: Markdown対応のレビュー本文（最大2000文字）
- **匿名投稿トグル**: 匿名での投稿を選択可能

### レビュー詳細画面
- **レビュー本文**: Markdown形式で整形表示
- **対象サービスのスコアリンク**: 関連するサービスのエシカルスコア画面への導線
- **参考になった/ならなかったボタン**: 有用性の投票
- **返信セクション**: スレッド形式のコメント欄

## ユーザーフロー

1. ユーザーがサービス詳細画面または代替提案画面から「レビューを見る」をタップする
2. システムが該当サービスに関するレビューをフィードに表示する
3. ユーザーがレビューを閲覧し、「参考になった」ボタンで有用性を投票する
4. ユーザーが自身の体験を共有したい場合、投稿ボタンをタップする
5. ユーザーがレビュータイプと対象サービスを選択する
6. ユーザーが評価と本文を入力し、匿名/実名を選択して投稿する
7. システムがレビューをモデレーション（AI自動 + 人的レビュー）にかけて公開する
8. 他のユーザーがレビューを閲覧し、返信やフィードバックを行う

## データモデル

### reviews テーブル
| カラム名 | 型 | 説明 |
|----------|------|------|
| id | UUID | 主キー |
| author_id | UUID | users.id への外部キー |
| review_type | ENUM('migration', 'service_review', 'discussion') | レビュータイプ |
| target_service_id | UUID | 対象サービスのservices.id |
| source_service_id | UUID | 乗り換え元サービス（migration時のみ） |
| rating | INTEGER | 星評価（1-5） |
| title | VARCHAR(200) | レビュータイトル |
| body | TEXT | レビュー本文（Markdown） |
| is_anonymous | BOOLEAN | 匿名投稿フラグ |
| status | ENUM('pending', 'published', 'rejected') | 公開ステータス |
| helpful_count | INTEGER | 「参考になった」数 |
| created_at | TIMESTAMP | 投稿日時 |
| published_at | TIMESTAMP | 公開日時 |

### review_replies テーブル
| カラム名 | 型 | 説明 |
|----------|------|------|
| id | UUID | 主キー |
| review_id | UUID | reviews.id への外部キー |
| author_id | UUID | users.id への外部キー |
| body | TEXT | 返信本文 |
| is_anonymous | BOOLEAN | 匿名投稿フラグ |
| created_at | TIMESTAMP | 投稿日時 |

### review_votes テーブル
| カラム名 | 型 | 説明 |
|----------|------|------|
| id | UUID | 主キー |
| review_id | UUID | reviews.id への外部キー |
| user_id | UUID | users.id への外部キー |
| vote_type | ENUM('helpful', 'not_helpful') | 投票タイプ |
| created_at | TIMESTAMP | 投票日時 |

### moderation_logs テーブル
| カラム名 | 型 | 説明 |
|----------|------|------|
| id | UUID | 主キー |
| review_id | UUID | reviews.id への外部キー |
| moderation_type | ENUM('auto', 'manual') | モデレーション種別 |
| result | ENUM('approved', 'rejected', 'flagged') | 判定結果 |
| reason | TEXT | 判定理由 |
| moderator_id | UUID | 人的モデレーター（manual時） |
| moderated_at | TIMESTAMP | 判定日時 |

## API設計

| メソッド | パス | 概要 |
|----------|------|------|
| GET | /api/reviews | レビューフィードを取得（フィルター・ページング対応） |
| GET | /api/reviews/:reviewId | レビュー詳細を取得 |
| POST | /api/reviews | 新規レビューを投稿 |
| GET | /api/reviews/service/:serviceId | 特定サービスのレビュー一覧を取得 |
| POST | /api/reviews/:reviewId/vote | レビューに投票する |
| GET | /api/reviews/:reviewId/replies | レビューへの返信一覧を取得 |
| POST | /api/reviews/:reviewId/replies | レビューに返信する |
| POST | /api/reviews/:reviewId/report | レビューを通報する |

## 非機能要件

- **モデレーション**: 投稿はAI自動モデレーション（不適切コンテンツ・スパム検出）を通過後に公開。フラグされた投稿は人的レビュー（24時間以内）
- **パフォーマンス**: レビューフィードの表示は500ms以内。ページネーションはカーソルベースで20件ずつ取得
- **スパム対策**: 同一ユーザーからの連続投稿制限（1日5件まで）。新規アカウントは最初の48時間は投稿にモデレーション必須
- **プライバシー**: 匿名投稿時は投稿者を一切特定できない設計。匿名投稿でもモデレーション違反時はアカウントに紐づけて対処可能
- **データ整合性**: サービスが削除された場合もレビューは保持（サービス名をスナップショットとして保存）
- **アクセシビリティ**: レビュー本文のMarkdownレンダリングはセマンティックHTMLで出力。スクリーンリーダーでの閲覧を保証
