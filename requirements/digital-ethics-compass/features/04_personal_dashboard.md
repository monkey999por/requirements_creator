# パーソナル倫理ダッシュボード

## 概要

個々のサービスのスコアを見るだけでは、自分のデジタルライフ全体がどの程度エシカルなのか把握しにくい。本機能は、ユーザーが登録した全サービスのエシカルスコアを集約し、「あなたのデジタルライフのエシカル度」を一目で把握できるダッシュボードを提供する。改善推移のトラッキングにより、サービス乗り換えの効果を実感できる。

## 画面構成

### ダッシュボードメイン画面
- **総合エシカルスコアカード**: ユーザーのデジタルライフ全体のスコア（0-100）とグレード（A-F）を大きく表示。前月比の変動を矢印で表示
- **軸別スコアサマリー**: 5軸それぞれの平均スコアを横棒グラフで表示
- **サービス別スコアランキング**: 登録サービスをスコア順に並べたリスト。最もスコアが低いサービスをハイライト
- **改善提案カード**: 「このサービスを乗り換えると+○点改善」という具体的な提案を最大3件表示
- **スコア推移グラフ**: 過去6ヶ月のスコア推移を折れ線グラフで表示

### インパクトレポート画面
- **月次レポート**: 今月のスコア変動要因サマリー（乗り換え効果、企業スコア変動等）
- **カテゴリ別分析**: SNS/EC/サブスク等のカテゴリ別にエシカル度を分析
- **業界平均との比較**: 同年代・同地域のユーザーとの匿名比較

## ユーザーフロー

1. ユーザーがホーム画面（ダッシュボード）を開く
2. システムが全登録サービスのスコアを集約し、総合エシカルスコアを表示する
3. ユーザーが軸別スコアサマリーで自分の弱点（最もスコアが低い軸）を確認する
4. ユーザーが改善提案カードから「代替を探す」をタップし、代替レコメンド機能に遷移する
5. ユーザーがスコア推移グラフで過去の改善状況を確認する
6. ユーザーがインパクトレポートで月次の変動要因を詳しく確認する

## データモデル

### user_dashboard_snapshots テーブル
| カラム名 | 型 | 説明 |
|----------|------|------|
| id | UUID | 主キー |
| user_id | UUID | users.id への外部キー |
| overall_score | INTEGER | 総合スコア（0-100） |
| overall_grade | CHAR(1) | 総合グレード（A-F） |
| privacy_avg | DECIMAL(5,2) | プライバシー軸平均 |
| labor_avg | DECIMAL(5,2) | 労働軸平均 |
| environment_avg | DECIMAL(5,2) | 環境軸平均 |
| tax_avg | DECIMAL(5,2) | 税務軸平均 |
| safety_avg | DECIMAL(5,2) | 安全性軸平均 |
| service_count | INTEGER | 登録サービス数 |
| snapshot_date | DATE | スナップショット日 |

### improvement_suggestions テーブル
| カラム名 | 型 | 説明 |
|----------|------|------|
| id | UUID | 主キー |
| user_id | UUID | users.id への外部キー |
| target_service_id | UUID | 改善対象サービスのservices.id |
| suggested_service_id | UUID | 提案代替サービスのservices.id |
| expected_improvement | INTEGER | 期待スコア改善幅 |
| priority | INTEGER | 提案優先順位 |
| created_at | TIMESTAMP | 作成日時 |

### monthly_reports テーブル
| カラム名 | 型 | 説明 |
|----------|------|------|
| id | UUID | 主キー |
| user_id | UUID | users.id への外部キー |
| year_month | CHAR(7) | 対象年月（YYYY-MM） |
| score_change | INTEGER | 前月比スコア変動 |
| summary_text | TEXT | AI生成の月次サマリー |
| factors | JSONB | 変動要因のリスト |

## API設計

| メソッド | パス | 概要 |
|----------|------|------|
| GET | /api/dashboard/me | ダッシュボードデータを取得 |
| GET | /api/dashboard/me/history | スコア推移データを取得 |
| GET | /api/dashboard/me/suggestions | 改善提案を取得 |
| GET | /api/dashboard/me/report/:yearMonth | 月次レポートを取得 |
| GET | /api/dashboard/me/comparison | 業界平均との比較データを取得 |

## 非機能要件

- **パフォーマンス**: ダッシュボードの初回表示は1秒以内。スコア推移グラフはクライアントサイドでレンダリングし、データ取得は500ms以内
- **データ集計**: ダッシュボードスナップショットは日次バッチで生成。リアルタイムの仮スコアも表示可能（「最終更新: ○時間前」を明記）
- **プライバシー**: 業界平均との比較は完全匿名の集計データのみ使用。個人が特定される粒度での比較は行わない（最低50人以上のグループで比較）
- **アクセシビリティ**: グラフ・チャートにはテキスト代替とデータテーブルを併設。色覚多様性に対応した配色を使用
- **オフライン対応**: 直近のスナップショットデータをローカルキャッシュし、オフラインでも最終閲覧時のダッシュボードを表示可能
