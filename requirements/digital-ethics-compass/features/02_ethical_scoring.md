# エシカルスコアリング

## 概要

企業やサービスの「倫理度」を客観的に評価することは、消費者が意識的な選択をするための基盤となる。しかし、既存のESG評価は投資家向けで一般消費者には理解しにくい。本機能は、プライバシー・労働・環境・税務・安全性の5軸で企業をスコアリングし、一般ユーザーにもわかりやすいビジュアル表現で提示する。AIがニュース・公開情報を自動分析し、スコアを継続的に更新する。

## 画面構成

### スコア概要画面
- **レーダーチャート**: 5軸（プライバシー/労働/環境/税務/安全性）のスコアをレーダーチャートで表示（0〜100点）
- **総合スコアバッジ**: A〜Fの6段階グレード表示（色分け: A=緑, F=赤）
- **スコア推移グラフ**: 過去12ヶ月のスコア変動を折れ線グラフで表示
- **根拠セクション**: 各軸のスコアの根拠となるニュース・情報ソースへのリンク

### スコア詳細画面（軸別）
- **スコア内訳**: 各軸の評価項目と個別スコア
- **ポジティブ要因/ネガティブ要因**: スコアに影響した具体的な事実をリスト表示
- **情報ソース一覧**: 評価に使用した記事・レポートへのリンク
- **AI要約**: スコアリング根拠をAIが平易な日本語で要約

## ユーザーフロー

1. ユーザーがサービス一覧から特定のサービスをタップする
2. システムがそのサービスの運営企業のエシカルスコアをレーダーチャートで表示する
3. ユーザーが特定の軸（例: プライバシー）をタップすると詳細画面に遷移する
4. システムがスコアの根拠となるニュース記事・公開情報を表示する
5. ユーザーがAI要約を読み、なぜそのスコアなのかを理解する
6. ユーザーが「スコアに異議がある」ボタンからフィードバックを送信できる

## データモデル

### companies テーブル
| カラム名 | 型 | 説明 |
|----------|------|------|
| id | UUID | 主キー |
| name | VARCHAR(200) | 企業名 |
| country | VARCHAR(100) | 本社所在国 |
| industry | VARCHAR(100) | 業種 |
| website_url | TEXT | 公式サイトURL |
| logo_url | TEXT | ロゴ画像URL |

### ethical_scores テーブル
| カラム名 | 型 | 説明 |
|----------|------|------|
| id | UUID | 主キー |
| company_id | UUID | companies.id への外部キー |
| privacy_score | INTEGER | プライバシースコア（0-100） |
| labor_score | INTEGER | 労働スコア（0-100） |
| environment_score | INTEGER | 環境スコア（0-100） |
| tax_score | INTEGER | 税務スコア（0-100） |
| safety_score | INTEGER | 安全性スコア（0-100） |
| overall_grade | CHAR(1) | 総合グレード（A-F） |
| scored_at | TIMESTAMP | スコアリング日時 |
| model_version | VARCHAR(20) | スコアリングモデルバージョン |

### score_evidences テーブル
| カラム名 | 型 | 説明 |
|----------|------|------|
| id | UUID | 主キー |
| ethical_score_id | UUID | ethical_scores.id への外部キー |
| axis | ENUM('privacy', 'labor', 'environment', 'tax', 'safety') | 対象軸 |
| impact | ENUM('positive', 'negative') | ポジティブ/ネガティブ |
| title | VARCHAR(300) | 根拠タイトル |
| summary | TEXT | AI生成の要約 |
| source_url | TEXT | 情報ソースURL |
| published_at | TIMESTAMP | ソースの公開日時 |

## API設計

| メソッド | パス | 概要 |
|----------|------|------|
| GET | /api/scores/:companyId | 企業のエシカルスコアを取得 |
| GET | /api/scores/:companyId/history | 企業のスコア推移を取得 |
| GET | /api/scores/:companyId/evidences | スコアの根拠情報を取得 |
| GET | /api/scores/:companyId/evidences/:axis | 特定軸の根拠詳細を取得 |
| POST | /api/scores/:companyId/feedback | スコアへのフィードバックを送信 |

## 非機能要件

- **パフォーマンス**: スコア表示は300ms以内（キャッシュ活用）。スコアリングバッチ処理は1企業あたり30秒以内
- **データ鮮度**: スコアは日次で再計算。重大なニュース（データ漏洩・訴訟等）は検知後1時間以内にスコアに反映
- **透明性**: スコアリングのアルゴリズムとウェイト配分を公開ドキュメントとして提供。ユーザーからのフィードバックを定期的にモデル改善に反映
- **バイアス対策**: スコアリングモデルは四半期ごとに第三者レビューを実施。地域・業種によるバイアスを補正する仕組みを導入
- **セキュリティ**: スコアデータの改ざん防止のため、スコアリング履歴は追記のみ（更新不可）の設計とする
