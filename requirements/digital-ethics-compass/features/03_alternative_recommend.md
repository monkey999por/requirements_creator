# 代替サービスレコメンド

## 概要

エシカルスコアが低いサービスを使い続ける理由の多くは「代わりが見つからない」ことにある。本機能は、ユーザーの利用パターン・価値観・必要機能を考慮して、よりエシカルな代替サービスをパーソナライズ提案する。単なる類似サービスの列挙ではなく、「乗り換えコスト」と「エシカル改善度」を天秤にかけた実用的な提案を行う。

## 画面構成

### 代替提案画面
- **対象サービスヘッダー**: 現在利用中のサービス名・エシカルスコア・グレードを表示
- **代替候補カードリスト**: 各候補のサービス名・エシカルスコア・機能カバー率・乗り換え難易度を表示。カードをタップすると詳細へ
- **フィルター**: 「スコア重視」「機能カバー重視」「乗り換え容易さ重視」の優先度スライダー
- **比較モード**: 2つのサービスを横並びで詳細比較するビュー

### 代替候補詳細画面
- **スコア比較チャート**: 現在のサービスと候補のレーダーチャートを重ねて表示
- **機能比較テーブル**: 主要機能ごとの対応状況を○/△/×で比較
- **乗り換えガイド**: データ移行手順、所要時間の目安、注意点
- **コミュニティ評価**: 実際に乗り換えたユーザーのレビュー（コミュニティレビュー機能と連携）

## ユーザーフロー

1. ユーザーがスコアの低いサービスの詳細画面で「代替を探す」をタップする
2. システムがユーザーの利用パターンと価値観設定に基づいて代替候補を算出する
3. システムが代替候補をスコア順にリスト表示する
4. ユーザーがフィルターで優先度を調整し、候補を絞り込む
5. ユーザーが候補カードをタップし、詳細比較画面を確認する
6. ユーザーが比較モードで2サービスを横並び比較する
7. ユーザーが乗り換えガイドを参照し、移行を開始する
8. 乗り換え完了後、システムがエシカルスコアの改善をダッシュボードに反映する

## データモデル

### service_features テーブル
| カラム名 | 型 | 説明 |
|----------|------|------|
| id | UUID | 主キー |
| service_id | UUID | services.id への外部キー |
| feature_name | VARCHAR(200) | 機能名 |
| feature_category | VARCHAR(100) | 機能カテゴリ |

### alternative_pairs テーブル
| カラム名 | 型 | 説明 |
|----------|------|------|
| id | UUID | 主キー |
| source_service_id | UUID | 元サービスのservices.id |
| alternative_service_id | UUID | 代替サービスのservices.id |
| feature_coverage_rate | DECIMAL(3,2) | 機能カバー率（0.00-1.00） |
| migration_difficulty | ENUM('easy', 'medium', 'hard') | 乗り換え難易度 |
| ethical_improvement | INTEGER | エシカルスコア改善幅 |
| updated_at | TIMESTAMP | 更新日時 |

### migration_guides テーブル
| カラム名 | 型 | 説明 |
|----------|------|------|
| id | UUID | 主キー |
| alternative_pair_id | UUID | alternative_pairs.id への外部キー |
| steps | JSONB | 移行手順（ステップ配列） |
| estimated_minutes | INTEGER | 所要時間（分） |
| warnings | TEXT[] | 注意事項 |

### user_value_preferences テーブル
| カラム名 | 型 | 説明 |
|----------|------|------|
| id | UUID | 主キー |
| user_id | UUID | users.id への外部キー |
| privacy_weight | DECIMAL(2,1) | プライバシー重視度（0.0-5.0） |
| labor_weight | DECIMAL(2,1) | 労働重視度 |
| environment_weight | DECIMAL(2,1) | 環境重視度 |
| tax_weight | DECIMAL(2,1) | 税務重視度 |
| safety_weight | DECIMAL(2,1) | 安全性重視度 |

## API設計

| メソッド | パス | 概要 |
|----------|------|------|
| GET | /api/alternatives/:serviceId | 指定サービスの代替候補を取得 |
| GET | /api/alternatives/:serviceId/:altServiceId | 2サービスの詳細比較データを取得 |
| GET | /api/alternatives/:serviceId/:altServiceId/guide | 乗り換えガイドを取得 |
| GET | /api/preferences/me | ユーザーの価値観設定を取得 |
| PUT | /api/preferences/me | ユーザーの価値観設定を更新 |

## 非機能要件

- **パフォーマンス**: 代替候補の算出は2秒以内。比較チャートのレンダリングは500ms以内
- **レコメンド品質**: 機能カバー率80%以上の代替候補を最低1件は提示することを目標とする。候補がない場合は「現在最適な代替が見つかりません」と正直に表示
- **パーソナライズ**: ユーザーの価値観ウェイトを反映したスコアリング。利用履歴から不要な機能を除外し、実質的なカバー率を算出
- **データ鮮度**: 代替ペアのマッピングは週次で更新。新サービスの登場を検知し、自動で候補に追加
- **アクセシビリティ**: 比較チャートにはテキスト代替を提供。スクリーンリーダーで比較内容を読み上げ可能にする
